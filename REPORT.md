
# Отчет по лабораторной работе №1: Анализ больших данных 

## Цель работы
Трансформация исходных сырых данных (10 CSV файлов по 1000 строк) в аналитическую модель данных типа «Снежинка» (Snowflake Schema) с нормализацией справочников до 3-й нормальной формы (3NF).

## Стек технологий
* PostgreSQL 15 (развертывание через Docker)
* SQL (DDL, DML)

## Структура проекта
```text
.
├── data/
│   ├── MOCK_DATA.csv
│   ├── MOCK_DATA (1).csv
│   └── ... (до 9)
├── sql/
│   ├── 01_init_source.sql       # Создание staging-таблицы и импорт сырых CSV-данных
│   ├── 02_ddl_snowflake.sql     # Создание аналитической схемы (Снежинка)
│   └── 03_dml_transform.sql     # ETL-скрипт: фильтрация дубликатов и распределение данных
└── docker-compose.yml           # Автоматизация развертывания БД
```

## Аналитическая модель данных (Snowflake Schema)
Реализована строго нормализованная архитектура.

**Таблица фактов:**
* `fact_sales` (10 000 строк) — метрики продаж и внешние ключи.

**Измерения (Dimensions):**
* `dim_date` — календарное измерение.
* `dim_customer` — покупатели.
* `dim_seller` — продавцы.
* `dim_store` — магазины.
* `dim_product` — товары.
* `dim_supplier` — поставщики.

**Суб-измерения (Sub-dimensions для нормализации):**
* `dim_country` — единый географический справочник для всех сущностей.
* `dim_pet_type` — виды питомцев.
* `dim_pet_breed` — породы питомцев.
* `dim_category` — товарные категории.
